What is F5A?

AI synthesis from all reviewed documentation — intended to guide Storybook and component layout.

Product

Fist of Five (code name fov, package name in app.json) is a real-time confidence-voting tool for facilitators. Participants hold up 1–5 fingers digitally; hosts see the spread instantly — the “raised hands in a room” metaphor. The core job is to answer “How confident is this audience, right now?” in under a minute.

Tagline from marketing docs: “Vote together. See the spread. Talk about what matters.” Positioning: fastest way for any facilitator to make audience confidence visible in real time — from scrum teams (~10) to conference halls (~10,000).

F5A’s role in the platform

F5A is the product surface — not marketing. It is an Expo universal app (iOS, Android, web export at app.fistoffive.co, which should be noindex per SEO docs).

  • Hosts primarily use the native mobile app — fast session creation, QR/link sharing, live tallies
  • Guests join via QR or invite link on web, iOS App Clip (vote-first, minimal chrome), or full app
  • No manual session code entry — join via QR or deep link only
  • Server-authoritative — session states and join/vote gates enforced by API, not UI guesswork

Core v1 loop

  1. Host signs in via Supabase (guest-host behind feature flag for now)
  2. Host creates or resumes a session → shareable link + QR
  3. Host explicitly “Enters session” before joining the realtime room (v4 invariant)
  4. Participants join and vote 1–5 on the current topic
  5. Tallies and per-person results stay visible throughout
  6. Optional ephemeral vote “reactions” — floating emoticons (Ably-powered, YouTube-lite; no video)
  7. Session lifecycle: CREATED → SCHEDULED → ACTIVE → ACTIVE_WITH_HOST_CONTROL → ENDED / EXPIRED

Explicit non-goals (current auth/onboarding scope)

  • Ably auth webhooks (reactions are separate from auth implementation)
  • Stripe billing UI (ADMIN subscription screen is documented but deferred)
  • host_intent OAuth split — dropped; unified sign-in only

Vote scale

Digital votes are 1–5 only. The physical Fist of Five practice sometimes uses fist=0; that is out of scope for the v1 product. Marketing copy should align over time.

Visual tone

Friendly, speed-first for onboarding and home shell: illustrated hero, dark bottom CTA card (Memoji-style). In-session UI stays clear and scannable — tallies and per-person results must never be buried.

Onboarding philosophy (host-first)

Mobile app skews host. First impression is action-first, not marketing: dominant “Go live” CTA; Join is secondary; resume active session when applicable. Dashboard is app home but not the focal point when no session is live. 2–3 onboarding slides max. Success metric: time-to-first-session.

Market audiences (from f5m content & research)

  • Scrum Masters — dev teams ~10, sprint/retro/planning confidence checks
  • Agile coaches & PI planning facilitators — scaled agile ceremonies
  • Executives / CEOs — all-hands ~100, strategic alignment visibility
  • Conference speakers — rooms up to 10,000+, QR on slides
  • Workshop facilitators — any group needing simultaneous honest signal

Use-case anchor

Agile Scrum / meeting facilitation. Scores below 4 signal a follow-up conversation opportunity. Simultaneous voting reduces dominant-voice bias and false consensus. Practitioner pain points (from research): slow manual polls, counting fingers at scale, performative voting, psychological safety concerns.
