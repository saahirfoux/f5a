Overview

Location: f5a/.cursor/rules/ · Multi-root workspace includes f5a, f5b, f5m

Cursor rules are short, always-on or file-scoped instructions for AI agents. f5a splits Expo-wide conventions (always apply) from Storybook-specific ones (apply only when relevant files are open) so agents are not overloaded on every edit.

Storybook vs normal app
bun run storybook → EXPO_PUBLIC_STORYBOOK_ENABLED=true → Metro bundles Storybook → _layout.tsx returns full-screen Storybook UI.
bun run start → Metro enabled: false → NativeTabs app shell only.

f5a-expo.mdc

alwaysApply: true — every f5a session

Bun / bunx (not npm / npx)

Purpose
Use the project package manager (bun.lock) for installs, scripts, and one-off CLIs.
What goes wrong without it
Mixed lockfiles, wrong dependency trees, or agents running npx expo start when scripts expect Bun.
Why a Cursor rule helps
Global habits default to npm; the rule makes Bun explicit for this repo only.

Routes under src/app/

Purpose
Expo Router screens live in src/app/, not a root-level app/ folder.
What goes wrong without it
New screens added at app/index.tsx are ignored; routing appears “broken” with no clear error.
Why a Cursor rule helps
Expo templates often use root app/; this project deliberately uses src/app/.

Path alias @/* → ./src/*

Purpose
Import app code via @/components/... per tsconfig.json paths.
What goes wrong without it
Deep relative imports (../../../components) that break when files move; inconsistent style.
Why a Cursor rule helps
Aliases are configured but not obvious from folder layout alone.

.rnstorybook/ not .storybook/

Purpose
React Native Storybook config lives in .rnstorybook/; web Storybook would use .storybook/.
What goes wrong without it
Agent initializes or edits web Storybook; RN stories never load or wrong tooling is installed.
Why a Cursor rule helps
Storybook 9/10 renamed the RN folder; most docs still say .storybook.

f5a-storybook.mdc

globs: stories, .rnstorybook/**, src/app/_layout.tsx, tsconfig.json, metro.config.js, babel.config.js

Dev workflow (bun run storybook vs start)

Purpose
Storybook only when EXPO_PUBLIC_STORYBOOK_ENABLED=true (via bun run storybook and variants).
What goes wrong without it
Agent runs bun run start and expects Storybook UI; or edits stories while Metro has Storybook disabled—stories missing, smaller bundle (~2MB vs ~4MB web export observed during setup).
Why a Cursor rule helps
This is the “front door” to Storybook; story-file rules do not say how to run the right mode.

Metro withStorybook + enabled flag

Purpose
metro.config.js wraps config with enabled: process.env.EXPO_PUBLIC_STORYBOOK_ENABLED === 'true'.
What goes wrong without it
Storybook always bundled in production/normal dev (bloat), or agent removes enabled and breaks require.context for stories.
Why a Cursor rule helps
One-line config is easy to “simplify” away; import path must stay @storybook/react-native/metro/withStorybook.

Babel: Reanimated plugin last

Purpose
react-native-reanimated/plugin is the last entry in babel.config.js plugins.
What goes wrong without it
Cryptic worklet/runtime errors in Storybook or animated screens after adding another plugin.
Why a Cursor rule helps
babel.config.js was added for Storybook; future Expo/React Compiler edits often reorder plugins.

NativeTabs — no /storybook route

Purpose
Storybook via env flag + early return in _layout.tsx; no src/app/storybook.tsx unless Stack refactor.
What goes wrong without it
Generic Expo+Storybook docs add app/storybook.tsx → Storybook inside tab chrome or forced layout rewrite.
Why a Cursor rule helps
Root layout uses NativeTabs, not Stack—contradicts most online examples.

Stories under src/ + sb-rn-get-stories

Purpose
Colocate *.stories.tsx; main.ts glob ../src/**/*.stories.*; regenerate index after glob changes.
What goes wrong without it
Stories in wrong folder; sidebar empty until bunx sb-rn-get-stories and Metro restart.
Why a Cursor rule helps
Story discovery is generated, not automatic on every save.

Entry: export default in .rnstorybook/index.ts

Purpose
Default-export Storybook UI for import in _layout.tsx; no registerRootComponent in layout-swap mode.
What goes wrong without it
Double entry points, blank screen, or Expo Router still owns root while Storybook also registers.
Why a Cursor rule helps
CLI templates use registerRootComponent; this project uses Expo Router swap instead.

Hooks before early return in _layout.tsx

Purpose
Call useColorScheme() (and all hooks) before the Storybook if return.
What goes wrong without it
Rules of Hooks violation—intermittent crashes or React errors in dev.
Why a Cursor rule helps
Early return for Storybook is correct pattern but easy to write hooks after it.

TypeScript: no @storybook/react-native/types

Purpose
Do not add that package to tsconfig.json compilerOptions.types (v10 does not ship it).
What goes wrong without it
tsc fails: cannot find type definition file.
Why a Cursor rule helps
Older guides reference the types package; generated storybook.requires.ts uses metro-env instead.

Cross-platform / Expo Go stories

Purpose
Prefer RN primitives and RN-web-compatible libs; avoid native-only modules in stories for Expo Go.
What goes wrong without it
Story works on simulator, crashes in Expo Go; agent assumes one environment.
Why a Cursor rule helps
Stories are where agents add imports; you target Go + iOS + Android + web.

Meta rules (documentation)

f5a-documentation.mdc

Purpose
New documentation under documentation/ is self-contained .html, not new .md.
What goes wrong without it
Multi-file Markdown that does not open cleanly when sharing one file; inconsistent doc format.
Why a Cursor rule helps
Agents default to .md; existing .md files remain valid but new guides should be shareable single files.

f5a-cursor-rules-maintain.mdc

Purpose
When .cursor/rules/ changes, update this HTML doc with purpose / failure mode / rationale per rule.
What goes wrong without it
Rules drift from docs; teammates and agents lose the “why” behind terse .mdc bullets.
Why a Cursor rule helps
Makes documentation a required part of the same change as the rule itself.

Adding a new rule

  1. Add .cursor/rules/your-rule.mdc with YAML frontmatter (description, globs or alwaysApply).
  2. Keep the .mdc short and actionable.
  3. Add a section to this file (or a new self-contained documentation/*.html) with Purpose, What goes wrong without it, and Why a Cursor rule helps for each convention.
  4. Add a row to the rules index table above.

Optional conventions documented elsewhere but not encoded as rules: cross-env in package.json scripts (already present); avoid default EXPO_PUBLIC_STORYBOOK_ENABLED in .env.local (discipline + dedicated storybook script).
