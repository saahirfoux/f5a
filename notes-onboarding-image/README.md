# Onboarding / poll hero screen — reference notes

Notes from analysis of the attached mock (real-time polling app, iPhone frame with Dynamic Island).

## What it is

- Mobile **onboarding or splash** screen for a **real-time polling** product.
- **Top ~60%**: light hero (`~#F9F9F9`), subtle grid, floating avatars + speech bubbles, central “stats” card.
- **Bottom ~40%**: dark sheet (`~#1C1C1C`), rounded top corners — **carousel-style** onboarding (dots + primary CTA).

## Visual language

- **Memoji-style** faces and pastel avatar rings → friendly, consumer/social tone.
- Speech bubbles with short reactions (“Yes!”, “No!”, “Sure!”) reinforce **live feedback**, not static marketing.

## Hierarchy

1. Faces + bubbles, then the white **summary card** (“You have 12 Active Poll”).
2. **Headline**: “All-in-one Real time Polling”.
3. **Body**: one line on instant audience feedback.
4. **Pagination dots** → multiple slides before commitment.
5. **Pill CTA** (`~#3B52FF`): “Get Started”.

## Typography & platform

- Clean sans-serif (Inter / SF Pro–like); strong white-on-dark in the bottom card.
- Status bar + Dynamic Island → **iOS** presentation (marketing mock or store-style asset).

## Copy / polish

- “12 Active Poll” → likely **“12 Active Polls”** (or “12 active polls”).
- “No Tanks!” → almost certainly **“No Thanks!”** unless intentional joke.
- “Real time” → often **“Real-time”** in product copy.

## UX

- **Get Started** sits low in the dark card; on device, add **safe-area** padding above the home indicator so the button does not feel cramped.
